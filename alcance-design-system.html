Como usar tokens
/* Importar variáveis globais */
:root {
  --color-blue: #58a6ff;
  --bg-base:    #0d1117;
}

/* Usar em componentes */
.meu-componente {
  background: var(--bg-card);
  color:      var(--text-primary);
  border:     1px solid var(--border);
}

Nunca use valores hexadecimais diretamente em componentes. Sempre referencie tokens para garantir consistência entre temas.

Buttons
Elementos de ação primária e secundária. 5 variantes semânticas em 3 tamanhos.
Variantes semânticas
Primary, Secondary, Success, Danger e Ghost. Cada variante comunica a intenção da ação — use Primary para ações principais, Danger para exclusões irreversíveis.
3 tamanhos (sm, md, lg)
Tamanho sm para ações compactas em tabelas ou toolbars. md é o padrão para formulários. lg para CTAs de destaque.
Suporte a ícones
Ícones podem ser adicionados antes ou após o label. Use ícones outline de 14–16px. O gap entre ícone e texto é controlado automaticamente pela classe .btn.
Estado desabilitado
Adicione o atributo disabled para bloquear interação. O botão recebe opacity: 0.4 e pointer-events: none automaticamente.

Fields & Inputs
Campos de texto com estados visuais, ícones e mensagens de feedback.
4 estados visuais
Default, Focus (borda azul + glow), Error (borda vermelha) e Disabled (opacidade reduzida). Cada estado comunica o contexto ao usuário sem depender apenas de cor.
Ícone prefixo
Use .field-input-wrap + .field-input-icon para adicionar um ícone à esquerda do campo. O padding é ajustado automaticamente.
Mensagens de feedback
Use .field-hint para textos de ajuda e .field-error-msg para erros de validação. Devem ser associados ao input via aria-describedby.
Suporte a textarea e select
A classe .field-input funciona em input, textarea e select. Para textarea use resize: vertical para permitir redimensionamento vertical.

Checkbox & Radio
Controles de seleção com estados visuais customizados e suporte a descrição secundária.
Checkbox — seleção múltipla
Permite selecionar zero ou mais opções de uma lista. O ícone de check é um SVG embutido — não depende de fonte de ícones ou imagens externas.
Radio — seleção única
Agrupe radios com o mesmo name para garantir seleção exclusiva. O estado marcado é controlado via CSS puro com o seletor :checked.
Descrição secundária
Adicione um elemento com .check-sub abaixo do label para fornecer contexto adicional sobre a opção. Útil em configurações e permissões.

Badges
Labels semânticos para status, categorias e estados. 7 variantes em formato pílula.
7 variantes semânticas
Green (ativo/sucesso), Blue (info/revisão), Red (erro), Yellow (aviso), Purple (beta), Coral (deprecated) e Ghost (neutro/rascunho). A escolha comunica contexto sem precisar de texto adicional.
Badge dot (status)
Adicione <span class="badge-dot"> dentro do badge para indicar estado em tempo real — "Online", "Running" ou "Pendente".
Inline em contextos
Badges são inline-flex e podem ser combinados com texto em tabelas, cards e sidebars. Use em combinação com avatares para indicar papel e status simultâneos.

Alert

4 variantes semânticas
Info (azul), Success (verde), Warning (amarelo) e Error (vermelho). A escolha da variante deve refletir o impacto real — Warning para algo que pode piorar, Error para algo que já falhou.
Título + descrição

Ícone SVG embutido
O ícone à esquerda reforça o tipo de mensagem visualmente. Use .alert-icon para manter o ícone alinhado ao topo quando o texto é longo.

Tabs
Alternância entre conteúdos relacionados com barra de navegação por abas.

Use .tab-count para exibir o número de itens na aba. Mantenha atualizado dinamicamente — contadores desatualizados confundem o usuário.

Collapse

Modal
Diálogos sobrepostos com overlay, animação de entrada e estrutura header/body/footer.
Overlay com blur
O overlay usa backdrop-filter: blur(4px) sobre o fundo escuro. Clicar no overlay fecha o modal — nunca bloqueie o fechamento por clique externo.
Animação de entrada
O modal abre com keyframe modalIn — scale 0.96 → 1 e translateY -8px → 0, duração 0.18s. Transmite aparição natural sem ser agressivo.
Estrutura semântica
Divida sempre em .modal-header, .modal-body e .modal-footer. O footer sempre deve ter ao menos um botão de cancelar ou fechar.

Tooltip
Informações contextuais exibidas ao passar o cursor sobre um elemento.
CSS puro — hover
O tooltip aparece com opacity: 1 no :hover do wrapper. Nenhum JavaScript necessário. Use pointer-events: none no tooltip para não interferir no mouse.
Seta indicadora
O pseudo-elemento ::after cria a seta triangular abaixo do tooltip usando o truque de borders transparentes. Sempre aponta para o elemento âncora.
Conteúdo textual simples
Mantenha o tooltip com texto curto — máximo uma linha. Para informações longas, use um Popover ou Modal. Tooltips devem complementar, não substituir labels.

Autocomplete
Campo de busca com lista de sugestões filtradas dinamicamente.
Filtragem em tempo real
Use o evento oninput para filtrar os itens da lista enquanto o usuário digita. Exiba apenas itens que correspondem ao valor atual do campo.
Abertura por foco
A lista abre no onfocus e fecha no onblur com timeout de 200ms para permitir o clique nos itens antes do fechamento.
Ícone no item
Cada item pode ter um ícone SVG de 14px à esquerda. Mantenha ícones consistentes com o tipo de dado — por exemplo, ícones de componente para busca de componentes.

Tables
Tabelas de dados com cabeçalho, hover por linha e suporte a ordenação.
Hover por linha
Linhas do tbody têm :hover com background sutil azul e text-primary para indicar selecionabilidade. Não use zebra striping — quebra a consistência do dark theme.
Colunas ordenáveis
Adicione .sortable ao <th> e o indicador de seta no texto para sinalizar ordenação. Implemente a lógica de sort no JavaScript da aplicação.
Célula primária
Use .primary na célula de identificação principal da linha (nome, ID, título). Recebe color: var(--text-primary) e font-weight: 500 para destacar.

Datatable
Tabela avançada com toolbar, busca em tempo real, seleção e paginação.
Busca em tempo real
O campo de busca filtra todas as colunas via textContent das linhas. A filtragem usa display: none nas linhas que não correspondem — rápido sem remover do DOM.
Seleção de linhas
Checkbox na primeira coluna permite seleção individual de linhas. Use o checkbox do cabeçalho para selecionar/deselecionar todas — implemente "select all" no JavaScript da aplicação.
Paginação
Botões de página com estado .active para a página atual. O contador "Mostrando X de Y" deve ser atualizado dinamicamente conforme filtro e página mudam.
Toolbar de ações
A toolbar contém o campo de busca à esquerda e botões de ação à direita (Exportar, Novo). Pode ser expandida com filtros adicionais usando dropdowns.

Footer
Rodapé em grid multi-coluna com logo, links organizados e copyright.
Grid 2fr + 1fr × 3
A primeira coluna é mais larga (2fr) e contém o logo e descrição do produto. As 3 colunas seguintes (1fr cada) agrupam links por categoria: Produto, Developers, Empresa.
Linha de copyright
O rodapé inferior tem border-top: 1px solid #21262d para separação sutil. Exibe copyright à esquerda e badge de versão + licença à direita.
Títulos em maiúsculas
Os títulos de coluna (.footer-col-title) são em 10.5px uppercase com letter-spacing 0.08em — consistente com os labels da sidebar e cabeçalhos de tabela.
